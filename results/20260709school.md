# 학교 서버 실행 실험 목록 (2026-07-09)

연구실 서버(GPU 0-3)는 현재 10D 최종모델 기반 W×H sweep + spatial/encoder ablation 11종을 실행 중이므로,
아래 실험들은 **학교 서버**에서 병렬로 진행한다. 실험 결과는 각
`checkpoints/tsem/<실험명>/results.json`(test acc/macro-F1/클래스별 지표/혼동행렬/Persist baseline 자동 기록)과
`logs/tsem/*.log`를 공유하면 된다.

1. **코드 동기화** — 오늘(07-09)까지 수정된 파일이 반드시 최신이어야 함:
   - `train_tsem.py` (CLI: `--stop_delta --seed --radius --raw_append --decomp_kernel --epochs --gpus --experiment --encoder_type --model` + early stopping + best.pt 재평가)

   - `configs/tsem_sage_10th_2.yaml` (**기준 config — 모든 실험의 베이스**)
   - `modules/tsem_eval.py`, `modules/tsem_instant_label.py`

공통 사항: 모든 실험은 `configs/tsem_sage_10th_2.yaml`(10D 최종모델: 라벨 B안 + multiloss+KL +
불확실성 가중치 + augmentation 1+2+3 + early stopping patience 50)을 베이스로 CLI 오버라이드만 사용.
1실험당 대략 epoch 60-65초 × 150-250 epoch(early stop) ≈ **3-5시간** (GPU 4장 DataParallel 기준).

## 확정 1 — Stop 정의 ±δ ablation (2 실험) — 입력값은 10차-2(10D)와 동일, δ만 변경

**목적**: 현재 채택된 stop ±2 다수결(δ=2, n=5)이 최적인지 검증. δ가 커질수록 flicker를 더 흡수하지만
lane_change 판정 구간과 겹칠 여지도 커지는 트레이드오프 (docs/TSEM_journal_design.html "stop ±δ ablation" 표 참조).
라벨 분포는 사전 실측됨: train stop 비율 δ=2: 39.24% / δ=3: 39.87% / δ=4: 40.34%.

```bash
python -u train_tsem.py --config configs/tsem_sage_10th_2.yaml --stop_delta 3 \
  --experiment tsem_sage_10d_stopdelta3 --gpus 0,1,2,3 \
  > logs/tsem/train_10d_stopdelta3.log 2>&1

python -u train_tsem.py --config configs/tsem_sage_10th_2.yaml --stop_delta 4 \
  --experiment tsem_sage_10d_stopdelta4 --gpus 0,1,2,3 \
  > logs/tsem/train_10d_stopdelta4.log 2>&1
```

**비교 기준**: δ=2 = 10차-2 (test acc 81.91 / macro-F1 81.95).
**주의**: δ가 다르면 정답 라벨 자체가 달라지므로 accuracy 직접 비교는 참고용 — 각자의 Persist baseline
(results.json에 자동 포함) 대비 격차로 비교할 것.

## 확정 2 — 10차-2 다중 seed → 평균·표준편차 (3 실험)

**목적**: 지금까지 전 실험이 seed 42 단일 실행. 저널 제출용 최종 모델(10D)은 평균±표준편차가 필요.
(seed 42 결과는 이미 있음 = 10차-2)

```bash
for SEED in 41 43 44; do
python -u train_tsem.py --config configs/tsem_sage_10th_2.yaml --seed $SEED \
  --experiment tsem_sage_10d_seed$SEED --gpus 0,1,2,3 \
  > logs/tsem/train_10d_seed$SEED.log 2>&1
done
```

**결과 정리**: 4개 seed(41/42/43/44)의 test acc·macro-F1 → mean±std. NAGraphSAGE 원 논문도 4-seed 보고 관례.

## 확정 3 — 극좌표 위치 polar 11D (1 실험) — 월드 절대좌표 제외, [ρ, sinθ, cosθ] 추가

**목적**: 10차 계열에서 확정된 "절대좌표 +3.9%p = 위치 암기"를 **이식 가능한 형태로 대체**하는 가설 검증.
raw position(x,z) 대신 로터리 중심 기준 [ρ, sinθ, cosθ]를 semantic 8D에 추가(11D) — 다른 로터리에서는
중심만 재측정하면 같은 의미로 재사용 가능. **이 실험이 raw 수준(81.9%)에 도달하면 "구조 참조 위치"가
논문의 핵심 novelty 용어가 된다.**

```bash
python -u train_tsem.py --config configs/tsem_sage_10th_2.yaml --raw_append polar \
  --experiment tsem_sage_polar_11d --gpus 0,1,2,3 \
  > logs/tsem/train_polar_11d.log 2>&1
```

**비교 기준**: 9차 semantic 8D(78.05) < polar 11D(?) vs 10차-2 pos 10D(81.91).

## (선택) 이웃 반경 radius ablation (2 실험)

**목적**: radius=20m 고정값 검증 (docs "radius ablation" 표의 10m/30m 행 채우기).
radius가 바뀌면 캐시가 새로 생성됨(키에 r 포함).

```bash

python -u train_tsem.py --config configs/tsem_sage_10th_2.yaml --radius 30.0 \
  --experiment tsem_sage_10d_r30 --gpus 0,1,2,3 > logs/tsem/train_10d_r30.log 2>&1
```

## (선택) Augmentation 단독 조합 (3 실험)

**목적**: 9차에서 검증된 1+2+3 조합 중 어느 augmentation이 기여의 대부분인지 분리
(docs "Augmentation 조합 ablation" 표의 "1만/2만/3만" 행).
CLI 오버라이드가 없으므로 `configs/tsem_sage_10th_2.yaml`을 복사해 augment 블록만 수정:

```yaml
# configs/school_aug_noise.yaml  — 1만: noise_std 1.0, 나머지 0
# configs/school_aug_nbr.yaml    — 2만: neighbor_dropout_p 0.1, 나머지 0
# configs/school_aug_frame.yaml  — 3만: frame_dropout_p 0.1, 나머지 0
```

```bash
python -u train_tsem.py --config configs/school_aug_noise.yaml \
  --experiment tsem_sage_10d_aug1only --gpus 0,1,2,3 > logs/tsem/train_10d_aug1only.log 2>&1
# aug2only, aug3only 동일 패턴
```

**비교 기준**: 없음=8차 상당(77.6) vs 1+2+3=10차-2(81.91).

## (보류·논의 필요) 다중 장소 학습

공업탑+DRIFT A~D 혼합 학습 → 미학습 장소(DRIFT E/I) 평가. 절대좌표가 원리적으로 무력화되는 무대라
semantic novelty를 복원하는 가장 근본적인 실험이지만, 장소별 held-out split 지원 코드가 아직 없어
(현재 split은 파일 내 시간 분할) 별도 구현 필요 — 연구실에서 준비 후 재논의.

## 결과 공유 방법

각 실험 완료 후 아래 두 가지를 공유:
1. `checkpoints/tsem/<실험명>/results.json` — 정량 지표 전부 포함
2. `logs/tsem/<로그명>.log`의 마지막 ~30줄 (best epoch, early stop 시점 확인용)

공유받으면 연구실 쪽에서 docs/TSEM_journal_design.html의 해당 표(stop ±δ, radius, augmentation 조합,
입력 조합, kernel)와 메모리에 기록·분석함.
