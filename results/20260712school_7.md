# 학교 서버(192.168.1.7) 실행 실험 목록 (2026-07-12)

연구실 서버(GPU 0-3)는 현재 STGCN(GPU0)·TGN(GPU1)·Transformer(GPU3) 비교실험 +
TSEM-SAGE ablation 3종(interleaved GPU0, position_only GPU1, edge-temporal GPU2)이 동시에
돌고 있어 자리가 없다. 원래 로컬에 **DCRNN → STGCN 종료 대기**, **Forecast-MAE → TGN 종료 대기**,
**SIMPL → edge-temporal 종료 대기**로 순차 실행 예약(`scripts/queue_dcrnn_forecastmae.sh`,
`scripts/queue_simpl_after_edgetemporal.sh`)이 걸려 있었으나, 대기 없이 바로 돌릴 수 있도록
**학교 서버에서 3종을 병렬로 실행**하는 것으로 변경한다. (로컬 예약은 본 문서 작성 후 해제함.)

## 0. 사전 준비

1. **코드 동기화** — 아래 어댑터/원본 폴더가 최신이어야 함:
   - `comparison/dcrnn_tsem/` (어댑터: `train_dcrnn_tsem.py`, `config.yaml`, `model.py`)
   - `comparison/DCRNN/` (원본 clone, 미수정)
   - `comparison/forecastmae_tsem/` (어댑터: `train_forecastmae_tsem.py`, `config.yaml`, `model.py`)
   - `comparison/forecast-mae/` (원본 clone, 미수정)
   - `comparison/simpl_tsem/` (어댑터: `train_simpl_tsem.py`, `config.yaml`)
   - `comparison/SIMPL/` (원본 clone, 미수정)
   - `modules/data_manager_tsem.py`, `modules/tsem_eval.py`, `modules/tsem_instant_label.py` (공통 평가/데이터 파이프라인)
2. **데이터 경로 확인**: 각 `comparison/*_tsem/config.yaml`의 `data.data_dir`가 로컬 서버 경로
   (`/home/oem/data/TII_data/`)로 되어 있음 — 학교 서버에서는 `/home/oem/yklee/data/`로 로컬 수정
   (커밋하지 말 것, `.claude/CLAUDE.md` 경고 참조).
3. conda env `tna_research` (torch 2.x + CUDA). 세 모델 다 Mamba 미사용 → mamba_ssm 불필요.
4. 캐시: 첫 실행 시 `cache/tsem/`에 W=10/H=10/stop_delta=2/radius=20 조합으로 자동 생성 — 세 실험이
   그래프 설정(`radius 20, K_max 6, K_max2 4`)이 동일하므로 캐시 1회 생성 후 공유됨.

공통 그래프/증강/손실 설정은 세 config가 이미 동일(§원칙 5, `comparison/README.md` 참조) —
model 블록만 모델별 고유. 평가는 전부 `modules/tsem_eval.py::evaluate_tsem` 재사용.

## 1. DCRNN (Li et al., ICLR 2018) ★★★

```bash
git pull origin main
CUDA_VISIBLE_DEVICES=0 python -u comparison/dcrnn_tsem/train_dcrnn_tsem.py \
  --config comparison/dcrnn_tsem/config.yaml \
  --experiment dcrnn_tsem_w10_h10 --gpus 0 \
  > logs/tsem/comparison_dcrnn_tsem_w10_h10.log 2>&1
```

## 2. Forecast-MAE (ICCV 2023) — 2단계(사전학습 → 미세조정) ★★★

```bash
git pull origin main
# 1) 사전학습 (레이블 미사용, 마스킹-복원 loss만)
CUDA_VISIBLE_DEVICES=1 python -u comparison/forecastmae_tsem/train_forecastmae_tsem.py \
  --config comparison/forecastmae_tsem/config.yaml --mode pretrain \
  --experiment forecastmae_tsem_w10_h10 --gpus 1 \
  > logs/tsem/comparison_forecastmae_tsem_pretrain.log 2>&1

# 2) 미세조정 (1단계 best 체크포인트 자동 로드)
CUDA_VISIBLE_DEVICES=1 python -u comparison/forecastmae_tsem/train_forecastmae_tsem.py \
  --config comparison/forecastmae_tsem/config.yaml --mode finetune \
  --experiment forecastmae_tsem_w10_h10 --gpus 1 \
  > logs/tsem/comparison_forecastmae_tsem_finetune.log 2>&1
```

멀티 GPU라도 이득 없음 경고가 뜨므로(`_warn_multi_gpu_no_gain`) 단일 GPU(`--gpus 1`)로 고정.

## 3. SIMPL (RA-L 2024) ★★★

```bash
git pull origin main
CUDA_VISIBLE_DEVICES=2 python -u comparison/simpl_tsem/train_simpl_tsem.py \
  --config comparison/simpl_tsem/config.yaml \
  --experiment simpl_tsem_w10_h10 --gpus 2 \
  > logs/tsem/comparison_simpl_tsem_w10_h10.log 2>&1
```

> `--gpus`는 학교 서버의 빈 GPU 번호로 조정. 위 0/1/2는 예시일 뿐 실제 서버 점유 상태를 `nvidia-smi`로
> 확인 후 배정할 것.

## 결과 공유 방법

각 실험 완료 후 아래 두 가지를 공유:
1. `checkpoints/comparison/<experiment>/results.json` — test acc/macro-F1/클래스별 지표/혼동행렬/
   Persist baseline 자동 포함
2. `logs/tsem/comparison_*.log`의 마지막 ~30줄 (best epoch, early stop 시점 확인용)

**82.03±0.14** (`comparison/README.md` §진행 상황 참조). 기존 완료분 CRAT-Pred(75.02/72.84),
HiVT(77.48/76.35), QCNet(77.97/76.96)과 같은 표에 이어서 기록.

# 학교 서버 실행 현황 (2026-07-12, 시작)

3종 전부 **병렬 실행 중**. 서버 = 4× RTX 2080 Ti(11GB), env `tna_research`(torch 2.7.0+cu126, CUDA).

## 사전 조치 (커밋 금지)
- **데이터 심링크**: 이 서버 데이터 폴더명이 `01_Gongeoptap`이나 `get_csv_files()`는
  `{data_dir}/Gongeoptap/*.csv`를 glob → `ln -s 01_Gongeoptap /home/oem/yklee/data/Gongeoptap`
  생성(CSV 11개 매칭 확인).
- **config `data_dir` 로컬 수정**: `comparison/{dcrnn,forecastmae,simpl}_tsem/config.yaml` 3개의
  `data_dir`를 `/home/oem/data/TII_data/` → `/home/oem/yklee/data/`로 변경(**커밋하지 않음**,
  `.claude/CLAUDE.md` data_dir 충돌 경고 준수).
- **원본 clone 미사용**: `comparison/{DCRNN,forecast-mae,SIMPL}/`(원본) 폴더는 이 서버에 부재하나,
  각 어댑터가 자체 `model.py`를 import(원본 clone 비의존)하므로 실행에 영향 없음.
- **공유 캐시 레이스 방지**: 세 config가 그래프/윈도우 설정 동일(W10/H10/r20/K6-4/δ2) → 캐시 키 동일.
  DCRNN을 먼저 띄워 캐시(train/val/test 3분할) 생성 완료 후 나머지 2종을 병렬 기동 → 무충돌 재사용 확인.
  - 캐시 키: `*_W10_H10_r20.0_K6-4_hybrid_tsem_f16ne_v4_tr0.7_vr0.15_58c806f17c`
  - 샘플 수: **train 575,686 / test 120,435** (train 라벨분포 stop 39.2% / LC 20.5% / normal 40.2%)
  - TSEM 클래스 가중치(power=0.7): stop=0.892 / lane_change=1.404 / normal=0.877

## 실행 명령 (GPU 배정)

| 실험 | GPU | 로그 | PID | 파라미터 |
|---|---|---|---|---|
| DCRNN | 0 | `logs/tsem/comparison_dcrnn_tsem_w10_h10.log` | 17121 | 190,659 |
| Forecast-MAE (pretrain→finetune 체인) | 1 | `..._forecastmae_tsem_{pretrain,finetune}.log` | 25580 | — |
| SIMPL | 2 | `logs/tsem/comparison_simpl_tsem_w10_h10.log` | 25685 | 488,035 (batch 2048) |

- Forecast-MAE는 `bash -c 'pretrain && finetune'` 체인으로 기동(pretrain best 체크포인트 자동 로드).
- 손실: 세 모델 공통 `L_focal + 0.1·KL(logits‖Uniform)` + stop_conf 불확실성 가중치.
- 학습: early_stop patience=50, grad_clip max_norm=5.0, best.pt 재로드 후 test 평가.

## 진행 상태 스냅샷 (기동 직후)
- DCRNN: Epoch 12 진행(~20 it/s, 1125 it/ep), GPU0 35%.
- Forecast-MAE pretrain: Epoch 13/50, recon_loss 하강 중, GPU1 37%.
- SIMPL: Epoch 4 진행(batch 2048, ~1.5 it/s, 282 it/ep), GPU2 93% / 5.7GB.
- 세 잡 모두 정상 학습(에러 없음). 완료 시 §결과 공유 방법대로 `results.json` + 로그 tail 기록 예정.

## 결과 (완료분 기록)

비교표(각자 라벨 기준, window+B안 δ=2). TSEM-SAGE 10D 최종 **81.97±0.14 / 82.03±0.14**.
기존 완료분: CRAT-Pred 75.02/72.84, HiVT 77.48/76.35, QCNet 77.97/76.96.

| 모델 | test acc | macro-F1 | Persist(acc/F1) | best ep | 비고 |
|---|---|---|---|---|---|
| **DCRNN** | **73.43** | **71.45** | 63.17 / 50.55 | 66 (ep116 early stop) | params 190,659 |
| **Forecast-MAE** | **76.09** | **74.65** | 63.17 / 50.55 | 92 (ep142 early stop) | params 305,731, pretrain 50ep→finetune |
| **SIMPL** | **75.20** | **73.71** | 63.17 / 50.55 | 107 (ep157 early stop) | params 488,035, batch 2048 |

**DCRNN 클래스별(test)**: stop P.780/R.937/F1.852 · lane_change P.632/R.636/F1.634 ·
normal P.732/R.597/F1.657. → LC/normal은 준수하나 acc 73.43은 현 비교군 중 최저(CRAT-Pred 하회).

**Forecast-MAE 클래스별(test)**: stop P.791/R.934/F1.857 · lane_change P.688/R.687/F1.687 ·
normal P.763/R.639/F1.696. → 2단계 사전학습 효과로 DCRNN 대비 전 클래스 개선(acc +2.66%p),
CRAT-Pred(75.02) 상회·HiVT(77.48) 하회 구간.

**SIMPL 클래스별(test)**: stop P.764/R.959/F1.850 · lane_change P.686/R.691/F1.689 ·
normal P.774/R.594/F1.672. → Forecast-MAE와 유사한 프로파일이나 acc 소폭 하회, CRAT-Pred(75.02) 근접 상회.

로그/체크포인트: `logs/tsem/comparison_{dcrnn,forecastmae,simpl}_tsem_*.log`,
`checkpoints/comparison/{dcrnn,forecastmae,simpl}_tsem_w10_h10/results.json`.

## 종합 (3종 완료, 2026-07-13)

전체 순위(acc): TSEM-SAGE 10D **81.97** ≫ QCNet 77.97 > HiVT 77.48 > **Forecast-MAE 76.09**
> **SIMPL 75.20** > CRAT-Pred 75.02 > **DCRNN 73.43**. macro-F1도 동일 경향.
세 신규 비교모델(DCRNN/Forecast-MAE/SIMPL) 모두 TSEM-SAGE 10D 대비 **−5.9~8.5%p** 열위로,
공업탑 로터리 예측에서 제안모델 우위를 재확인. 세 모델 공통 약점은 **normal recall**(.59~.64)로,
"1초 뒤 출발" anticipation 난제(§확정 발견 5)가 비교군 전반에 공통됨.
