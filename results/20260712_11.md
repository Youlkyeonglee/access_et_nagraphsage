# 학교 서버(192.168.1.11) 실행 실험 목록 (2026-07-12) — 20260712 추가 실험계획

로컬 서버(GPU 0-3)는 현재 STGCN(GPU0 자리→interleaved로 교체됨)·TGN(GPU1→position_only)·
edge-temporal(GPU2)·Transformer(GPU3)로 이미 꽉 차 있어, §실험현황 "20260712 추가 실험계획"
(6D invariant가 HiVT·QCNet에 뒤지는 격차를 좁히기 위한 4개 조치 + 조합 실험)은 이 서버에서
진행한다. 6D invariant 기준선(76.21%/74.78%)은 로컬에서 이미 완료돼 있으니 재실행 불필요.

## 0. 사전 준비

1. **코드 동기화** — 아래 파일이 반드시 최신(2026-07-12 커밋)이어야 함:
   - `models/tsem_semantic_derivation.py` — `variant='invariant_rot'`(8D: v,a,j,ω,d_lat,κ,
     cos_rot,sin_rot) + `LearnableSemanticResidual` 클래스 신설
   - `models/tsem_sage.py` — `TSEMSAGE.__init__`에 `learnable_residual_dim: int = 0` opt-in
     파라미터 추가(0이면 기존 동작과 완전 동일)
   - `models/temporal_encoder.py` — `encoder_type='transformer'` 신규 구현(2-layer
     `nn.TransformerEncoder`, sinusoidal PE, causal mask — `comparison/transformer_tsem`과
     동일 패턴)
   - `train_tsem.py` — `build_model()`이 `learnable_residual_dim`을 config에서 읽어 전달하도록
     배선
   - `modules/data_manager_tsem.py`, `modules/tsem_eval.py`, `modules/tsem_instant_label.py`
     (공통 데이터/평가 파이프라인, 변경 없음이지만 최신 상태 확인)
2. **데이터 경로 확인**: 각 config의 `data.data_dir`가 로컬 서버 경로
   (`/home/oem/data/TII_data/`)로 커밋돼 있음 — 이 서버 실제 경로(예:
   `/home/oem/yklee/data/` 등, `.claude/CLAUDE.md` "학교 서버" 항목 참조)로 로컬 수정만 하고
   **커밋하지 말 것**.
3. conda env `tna_research` (torch 2.x + CUDA). 전부 Mamba 미사용 → `mamba_ssm` 불필요.
4. **캐시**: W=10 실험 9개는 캐시가 W10/H10/radius20/K6-4 조합으로 로컬과 동일 키라 이 서버에서
   최초 1회 생성(~10-15분) 후 9개가 전부 공유한다. W=20/W=30 실험 4개는 각각 별도 캐시가
   새로 생성된다(캐시 키에 W가 포함되므로 자동 분리, 서로 안전).
5. **스모크테스트 검증 완료(로컬, 2026-07-12)**: 아래 18개 config 전부 랜덤 텐서로
   forward+backward+NaN 확인을 이미 통과했다 — 이 서버에서 별도 스모크 없이 바로 실행해도 된다.

공통 data/graph/train/loss/augment 설정은 전부 `configs/tsem_sage_invariant_6d.yaml`(6D
invariant 기준, 76.21%/74.78%)과 동일 — model 블록만(semantic_variant/hidden_dim/d_e/
encoder_type/learnable_residual_dim/W) 실험마다 다르다. 평가는 전부
`modules/tsem_eval.py::evaluate_tsem` 재사용.

## 1. ①②③ 단독 실험 (W=10 그대로, 캐시 공유) ★★★

```bash
git pull origin main

# ① 상대 heading 회전량 추가 (6D→8D)
CUDA_VISIBLE_DEVICES=0 python -u train_tsem.py \
  --config configs/tsem_sage_invariant_8d_rot.yaml --gpus 0 \
  > logs/tsem/train_invariant_8d_rot.log 2>&1

# ② 학습 가능한 잔차 브랜치 (speed,accel 기반 4D 추가)
CUDA_VISIBLE_DEVICES=1 python -u train_tsem.py \
  --config configs/tsem_sage_invariant_6d_learnable.yaml --gpus 1 \
  > logs/tsem/train_invariant_6d_learnable.log 2>&1

# ③ 용량 확대 (hidden_dim 128→192)
CUDA_VISIBLE_DEVICES=2 python -u train_tsem.py \
  --config configs/tsem_sage_invariant_6d_h192.yaml --gpus 2 \
  > logs/tsem/train_invariant_6d_h192.log 2>&1

# ③ 용량 확대 (hidden_dim 128→256)
CUDA_VISIBLE_DEVICES=3 python -u train_tsem.py \
  --config configs/tsem_sage_invariant_6d_h256.yaml --gpus 3 \
  > logs/tsem/train_invariant_6d_h256.log 2>&1
```

## 3. ①+②③④ 조합 실험 (①의 8D를 기본 입력으로) ★

①(cos_rot,sin_rot 2채널 추가)이 만드는 새 신호가 ②③④ 각각의 효과와 누적되는지 보는
후속 실험 — 1·2번 결과가 유의미한 조합부터 우선 진행해도 된다.

```bash
git pull origin main

# ①+② (8D + 학습형 잔차 4D = 12D)
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_learnable.yaml --gpus 1 \
  > logs/tsem/train_invariant_8d_rot_learnable.log 2>&1

# ①+③ (8D, hidden_dim 192 / 256)
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_h192.yaml --gpus 2 \
  > logs/tsem/train_invariant_8d_rot_h192.log 2>&1
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_h256.yaml --gpus 3 \
  > logs/tsem/train_invariant_8d_rot_h256.log 2>&1

# ①+④ (8D, W×encoder 그리드)
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_w10_transformer.yaml --gpus 0 \
  > logs/tsem/train_invariant_8d_rot_w10_transformer.log 2>&1
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_w20_gru.yaml --gpus 1 \
  > logs/tsem/train_invariant_8d_rot_w20_gru.log 2>&1
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_w20_transformer.yaml --gpus 2 \
  > logs/tsem/train_invariant_8d_rot_w20_transformer.log 2>&1
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_w30_gru.yaml --gpus 3 \
  > logs/tsem/train_invariant_8d_rot_w30_gru.log 2>&1
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_w30_transformer.yaml --gpus 0 \
  > logs/tsem/train_invariant_8d_rot_w30_transformer.log 2>&1
```

> `--gpus`는 이 서버의 빈 GPU 번호로 조정. 위 번호는 예시일 뿐 실제 점유 상태를 `nvidia-smi`로
> 확인 후 배정할 것. 파라미터 규모가 크지 않아(최대 675K) 1 config당 GPU 1개면 충분.

## 4. ①+②+③ 삼중 조합 — ⏳ 조건부, ②단독 결과 확인 후에만 실행

②(학습형 잔차)가 위 1번에서 유의미한 개선을 보일 때만 실행한다. **②단독 결과가 6D
invariant(76.21%/74.78%) 대비 뚜렷한 개선이 없으면 이 두 config는 실행하지 말 것** —
효과 없는 조치를 삼중 조합에 섞으면 무엇이 기여했는지 해석할 수 없다(§하네스 원칙
"한 번에 하나씩만 바꿔 격리").

```bash
git pull origin main

python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_learnable_h192.yaml --gpus 2 \
  > logs/tsem/train_invariant_8d_rot_learnable_h192.log 2>&1
python -u train_tsem.py --config configs/tsem_sage_invariant_8d_rot_learnable_h256.yaml --gpus 3 \
  > logs/tsem/train_invariant_8d_rot_learnable_h256.log 2>&1
```

스모크테스트(랜덤 텐서 forward+backward+NaN 확인)는 이미 통과했다 — params 388,027 / 681,147.

## 결과 공유 방법

각 실험 완료 후 아래 두 가지를 공유:
1. `checkpoints/tsem/<experiment>/results.json` — test acc/macro-F1/클래스별 지표/혼동행렬/
   Persist baseline 자동 포함
2. `logs/tsem/train_*.log`의 마지막 ~30줄 (best epoch, early stop 시점 확인용)

**비교 기준**:
| 모델 | test acc | test macro-F1 |
|---|---|---|
| Persist baseline | 63.17% | 50.55% |
| CRAT-Pred | 75.02% | 72.84% |
| **6D invariant (기준선, 로컬 완료)** | **76.21%** | **74.78%** |
| HiVT | 77.48% | 76.35% |
| QCNet | 77.97% | 76.96% |
| 10D 최종모델(10차-2, 비교 상한) | 81.91~81.97% | 81.95~82.03% |

①②③④ 각 단독/조합 실험이 6D invariant(76.21%/74.78%)를 얼마나 넘어서는지, 특히 HiVT
(77.48%)·QCNet(77.97%)을 넘어서는 조합이 있는지가 핵심 관찰 포인트.

조합 8 + ①+②+③ 삼중 조합 2, 마지막 2개는 조건부) — 6D invariant GRU(W=10) 기준선 자체는
로컬에서 이미 완료돼 재실행 불필요. 상세 배경·가설은 `docs/TSEM_journal_design.html`
