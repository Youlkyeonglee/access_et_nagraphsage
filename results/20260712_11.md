## 0. 사전 준비

1. **코드 동기화** — 아래 파일이 반드시 최신(2026-07-12 커밋)이어야 함:

     파라미터 추가(0이면 기존 동작과 완전 동일)

     배선
   - `modules/data_manager_tsem.py`, `modules/tsem_eval.py`, `modules/tsem_instant_label.py`
     (공통 데이터/평가 파이프라인, 변경 없음이지만 최신 상태 확인)
2. **데이터 경로 확인**: 각 config의 `data.data_dir`가 로컬 서버 경로
   (`/home/oem/data/TII_data/`)로 커밋돼 있음 — 이 서버 실제 경로(예:
   `/home/oem/yklee/data/` 등, `.claude/CLAUDE.md` "학교 서버" 항목 참조)로 로컬 수정만 하고
   **커밋하지 말 것**.
3. conda env `tna_research` (torch 2.x + CUDA). 전부 Mamba 미사용 → `mamba_ssm` 불필요.
4. **캐시**: W=10 실험 9개는 캐시가 W10/H10/radius20/K6-4 조합으로 로컬과 동일 키라 이 서버에서

## 예약 — §2 ④ W×encoder 그리드 (2026-07-12 21:15 예약, §1 종료 후 자동 실행)

메모리 인지형 런처(`scratchpad/run_s2_grid.sh`, PID 백그라운드)를 걸어둠. §1이 GPU를 비우면
(free≥21GB) 아래 **우선순위(메모리 오름차순)**로 자동 투입하며, 각 config를 단일 GPU로 띄워
**Epoch 1 통과=메모리 fit**을 확인한다. `out of memory`가 뜨면 kill 후 그 GPU를 다음 config에
양보하고, 해당 config를 **§2-deferred(다중 GPU 후보)**로 기록한다.

**메모리 예측 근거**: §1의 W10-gru가 ~16GB. W가 커지면 시퀀스 활성값이 늘고(GRU ~선형),
transformer는 attention이 O(W²)라 같은 W에서 더 무겁다. → 초과 위험 오름차순.

**캐시**: W10-transformer는 §1의 W10 캐시(`58c806f17c`, warm) 재사용. W20/W30은 각각 신규
캐시를 1회 빌드(~10-15분). pair(gru/transformer)가 같은 캐시 키를 공유하므로 **gru를 먼저
Epoch1까지 도달시킨 뒤 transformer를 띄워** 콜드 빌드 레이스를 방지한다(런처가 순차 보장).

| 순위 | 실험 | config | W/encoder | 캐시 | 예상 단일GPU 메모리 |
|---|---|---|---|---|---|
| 1 | W10-transformer | `..._6d_w10_transformer.yaml` | 10/tf | warm(재사용) | ~16–18GB ✅ 안전 |
| 2 | W20-gru | `..._6d_w20_gru.yaml` | 20/gru | 신규 빌드 | ~24GB 근처 ⚠️ |
| 3 | W20-transformer | `..._6d_w20_transformer.yaml` | 20/tf | warm(2번이 빌드) | 24GB 경계 ⚠️ |
| 4 | W30-gru | `..._6d_w30_gru.yaml` | 30/gru | 신규 빌드 | 24GB 초과 가능 ⛔ |
| 5 | W30-transformer | `..._6d_w30_transformer.yaml` | 30/tf | warm(4번이 빌드) | 24GB 초과 유력 ⛔ |

> 예측은 투입 순서용일 뿐, **실제 fit/초과 판정은 런처가 Epoch1/OOM으로 실측**한다.

**초과 config 처리(다중 GPU, 미뤄둠)**: 단일 24GB를 넘는 config는 `logs/tsem/s2_deferred.txt`에
아래 형태로 기록된다 — train_tsem.py는 `device_count>1`이면 자동 `nn.DataParallel`로 배치를
분할하므로 GPU 2장을 주면 batch 4096→2048/GPU로 쪼개져 실행 가능:
```bash
CUDA_VISIBLE_DEVICES=a,b python -u train_tsem.py --config configs/<cfg>.yaml > logs/tsem/<cfg>.log 2>&1 &
```
