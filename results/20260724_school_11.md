# 학교 서버 실행 실험 목록 (2026-07-24) — 위치암기 입력-통제 ablation (HiVT/QCNet + pos/polar)

baseline(HiVT-adapted, QCNet-adapted)이 원래 절대좌표를 안 쓰는 회전-불변 설계라, TSEM-SAGE
10D/11D와 직접 비교하면 "TSEM-SAGE의 우위가 구조 때문인지 site-specific 신호 때문인지"가
분리되지 않는다는 문제(§비교연구 논의, `docs/TSEM_journal_design.html:1710-1712`, `:2467`
"여전히 옵션으로 남아있다")를 해소하기 위해, 같은 site-specific 신호(raw position / 로터리
중심 기준 polar)를 HiVT/QCNet에도 opt-in으로 주입할 수 있게 `comparison/hivt_tsem/model.py`,
`comparison/qcnet_tsem/model.py`에 `raw_append: 'none'|'pos'|'polar'` 옵션을 추가했다
(기본값 'none'은 기존 동작과 100% 동일). 로컬에서 스모크 테스트(shape/param 수 확인, 기존
회귀테스트 `test_model.py` 전부 PASS, HiVT `polar` 모드 실제 데이터 학습 진행 확인) 완료.

## 1. 실행 명령 (GPU 4개 있으면 병렬, 없으면 순차)

```bash
git pull origin main

# HiVT-adapted + raw position 2채널(10D와 동일 신호)
CUDA_VISIBLE_DEVICES=0 python -u comparison/hivt_tsem/train_hivt_tsem.py \
  --config comparison/hivt_tsem/config_pos.yaml \
  > logs/tsem/train_hivt_pos.log 2>&1

# HiVT-adapted + polar(로터리 중심 기준 [ρ,sinθ,cosθ], 11D와 동일 신호)
CUDA_VISIBLE_DEVICES=1 python -u comparison/hivt_tsem/train_hivt_tsem.py \
  --config comparison/hivt_tsem/config_polar.yaml \
  > logs/tsem/train_hivt_polar.log 2>&1

# QCNet-adapted + raw position 2채널
CUDA_VISIBLE_DEVICES=2 python -u comparison/qcnet_tsem/train_qcnet_tsem.py \
  --config comparison/qcnet_tsem/config_pos.yaml \
  > logs/tsem/train_qcnet_pos.log 2>&1

# QCNet-adapted + polar
CUDA_VISIBLE_DEVICES=3 python -u comparison/qcnet_tsem/train_qcnet_tsem.py \
  --config comparison/qcnet_tsem/config_polar.yaml \
  > logs/tsem/train_qcnet_polar.log 2>&1
```

> `--gpus` 인자는 train_tsem.py 계열 argparse에 없음(붙이면 "unrecognized arguments"로 즉사) —
> `CUDA_VISIBLE_DEVICES=N`으로 GPU 1장씩 고정할 것(device_count>1이면 자동 `nn.DataParallel`이
> 걸려 고정 안 하면 다른 job과 GPU를 나눠 쓰다 충돌).

## 비교 기준

| 모델 | test acc | test macro-F1 | 비고 |
|---|---|---|---|
| HiVT-adapted (원본, raw_append=none) | 77.48% | 76.35% | 회전-불변, 절대좌표 미사용 |
| QCNet-adapted (원본, raw_append=none) | 77.97% | 76.96% | 회전-불변, 절대좌표 미사용 |
| HiVT-adapted + pos | **79.59%** | 79.11% | 이번 실험 (+2.11/+2.76) |
| HiVT-adapted + polar | **80.51%** | 80.40% | 이번 실험 (+3.03/+4.05) |
| QCNet-adapted + pos | **80.45%** | 80.21% | 이번 실험 (+2.48/+3.25) |
| QCNet-adapted + polar | **81.36%** | 81.49% | 이번 실험 (+3.39/+4.53) |
| TSEM-SAGE 10D(10차-2, 최종 채택) | 81.91~81.97% | 81.95~82.03% | site-specific 신호 포함 |
| TSEM-SAGE 11D(polar, "구조 참조 위치") | 81.82% | — | site-specific 신호 포함(이식 가능 버전) |

**관찰 포인트**: baseline+pos/polar가 원본(77.48/77.97) 대비 얼마나 오르는지, TSEM-SAGE
10D/11D(81.91~81.97/81.82)에 얼마나 가까워지는지 — 많이 오르면 "위치암기는 구조와 무관한
범용 효과", 별로 안 오르면 "TSEM-SAGE 구조가 위치 신호를 특히 잘 활용" 쪽으로 해석.

### 최종 결과 (2026-07-24 완료, 로컬 서버 GPU0-3)

| 실험 | best epoch | test acc | test macro-F1 | vs 원본(none) |
|---|---|---|---|---|
| HiVT + pos | 122 | 79.59% | 79.11% | +2.11 / +2.76 |
| HiVT + polar | 267 | 80.51% | 80.40% | +3.03 / +4.05 |
| QCNet + pos | 59 | 80.45% | 80.21% | +2.48 / +3.25 |
| QCNet + polar | 54 | 81.36% | 81.49% | +3.39 / +4.53 |

**해석 — 위치 신호는 구조와 무관한 범용 효과:**
- pos/polar 주입만으로 HiVT/QCNet가 **+2~3.4%p 크게 상승** → 절대좌표/구조참조 위치는 특정
  아키텍처 전용 이득이 아니라 **어떤 백본에도 이식되는 범용 site-specific 신호**임이 확인됨.
- **polar > pos** (HiVT 80.51 vs 79.59, QCNet 81.36 vs 80.45): "구조 참조 위치(로터리 중심 기준
  [ρ,sinθ,cosθ])"가 raw 절대좌표보다 나음 — TSEM-SAGE에서 관찰된 11D>10D 경향과 방향 일치.
- **QCNet+polar(81.36)이 TSEM-SAGE 11D(81.82)·10D(81.91~81.97)에 근접**(격차 −0.46~−0.61%p,
  noise 0.14보다는 크나 소폭). → 위치 신호를 주면 baseline이 TSEM-SAGE에 거의 따라붙음.

**논문 함의**: TSEM-SAGE의 HiVT/QCNet 대비 우위(원본 기준 +4%p대)는 **대부분 site-specific 위치
신호에서 왔고, 구조 자체의 순수 기여는 작다(QCNet+polar 대비 +0.5%p 내외).** "위치암기는 구조와
무관"이라는 관찰 포인트 쪽으로 결론 — `docs/TSEM_journal_design.html` §비교연구 논의(§1710·§2467
"옵션으로 남아있다")를 이 결과로 갱신 필요.
