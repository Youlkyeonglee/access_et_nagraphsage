시계열 성능 향상 탐색 + 메인 데이터셋 DRIFT 전환

ET-NAGraphSAGE · 2026-07-04 갱신 · 교수님 보고용 — 왜 메인 데이터셋을 공업탑→DRIFT로 바꾸는가
핵심 보고 (2026-07-04):
  • 공업탑(로터리)에서는 차량 간 상호작용 신호 자체가 없어 edge(관계)가 죽었다 — 4중 음성 검정으로 확증. 방법·설계가 아니라 데이터의 한계.
  • 메인 데이터셋을 DRIFT(밀집 다차로, 6개 도로 A~E·I)로 전환하면 상호작용이 강하고, edge가 6개 도로 전부에서 차선변경을 유의미하게 향상(+2.0~+9.5%p, p=0.0002).
  • ⇒ 우리가 원한 spatial-temporal + edge novelty를 정당하게 살릴 무대는 DRIFT. 현재 본 모델(2-hop 엣지-시계열 GNN)을 DRIFT 전체에 적용해 학습 중.

0. 메인 데이터셋 전환 — 공업탑 → DRIFT (교수님 보고)

0-1. 왜 바꾸는가 — 공업탑은 상호작용이 없는 데이터

공업탑에서 edge(이웃 관계)가 기여하지 못한 이유를 4가지 독립 검정으로 규명했다. 모두 "이웃 신호 없음"으로 일치한다.

진단: 공업탑은 저밀도 로터리 + magnitude-only 가속도라 측정 가능한 차량 간 상호작용이 거의 없다. edge가 배울 신호 자체가 없으므로, edge novelty를 이 데이터로 검증하는 것은 구조적으로 불가능하다.

0-3. 지금 진행 중 — 본 모델을 DRIFT 전체에 적용

A/B 경량 검증(edge가 산다)을 확인한 뒤, 본 2-hop 엣지-시계열 GNN(ET-NAGraphSAGE)을 DRIFT 전체(99파일)에 적용해 학습 중이다. 설계 요점:

  • 장면 불변 설계: 도로 A~I의 좌표계가 다르므로, 프레임 t의 ego 위치 기준으로 모든 노드 position을 상대화(ego_relative) → 절대위치 과적합 제거. 엣지는 상대 kinematic(heading gap)이라 원래 장면 불변.
  • 장면별 시간분할: 각 파일=한 장면 → 파일별 프레임 시간분할로 장면 간 누수 없음.
  • 정규화: DRIFT 추적 노이즈(NaN/극단 가속도) 대응 위해 nan 제거 + clip + z-score 표준화 적용.
  • C-ablation 동시 실행: temporal_target = both / node / edge 세 실험을 GPU 3장에서 병렬 → 본 모델에서 edge 시계열의 실제 기여를 직접 절제 측정.

보고 요지: "edge가 안 된다"는 결론은 공업탑 데이터 한정이며, 원래 목표인 spatial-temporal + edge novelty는 상호작용이 있는 DRIFT를 메인 데이터셋으로 삼으면 정당하게 성립한다. 아래 1~3장(공업탑 기반 비-edge 아이디어)은 공업탑을 계속 쓸 경우의 대안이며, DRIFT 전환이 본선이다.

— 이하: 공업탑 기준 이전 분석 (참고) —

요약 (공업탑 한정):
  • 정식 2×2 절제에서 엣지(관계)는 강한 ego 표현에 도움 안 됨이 확인됐다(별도 보고서). — 단, 위에서 밝혔듯 공업탑에 상호작용이 없기 때문.
  • 단, HD맵 도로정렬 피처를 world에 *추가(augment)*하면 유의미하게 향상(탐지 +1.5%p, 예측 +1.0%p, p<0.001). ⇒ 맵은 노드(ego) 표현으로는 가치 있음.
  • 따라서 (공업탑을 유지할 경우) 성능 향상의 축은 "이웃(edge)"이 아니라 "ego 시계열을 더 잘 인코딩". 아래 비-edge 아이디어 제안.

2. 비-edge 시계열 성능 향상 아이디어

ego 시계열이 지배적이므로, "이웃"이 아니라 "ego의 시간 동역학을 더 잘 뽑는" 방향들. 유망순.

#아이디어왜 효과 있나비용
1 추천시계열 인코더 교체 (GRU→Mamba/Transformer/TCN)ego 동역학이 핵심이니 인코더가 곧 성능. Mamba는 긴 시퀀스서 강점 — 원래 관심 축을 깨끗한 ego 무대에서 검증낮음
2 추천다중 스케일 시계열 (짧은창+긴창 병렬 인코딩)차선변경은 지속시간 편차 큼 — 세밀한 순간 + 긴 맥락 동시 포착중
3시간 어텐션 풀링 (마지막 은닉 대신 프레임 어텐션)기동 순간에 집중 → 정확도↑ + 해석가능성(언제 판단했나, 논문 그림)낮음
4고차 시간 미분 피처 (jerk, 반경가속도 ΔΔρ, 궤적 곡률)차선변경은 특유의 횡방향 가·감속 서명. 도로정렬 augment의 확장낮음
5궤적 예측 보조과제 (미래 ρ/위치 forecasting + 분류 멀티태스크)시계열 인코더가 더 풍부한 동역학 학습(자기지도적)중
6다지평 공동 학습 (H=6/15/30 동시 예측)공유 시계열 표현 정규화, 지평 일반화낮음
7시간 워핑 증강 (time-warp)기동 지속시간 다양성 확보 → 소수(차선변경) 일반화낮음
8입력 창 확장 (현재 10프레임 → 20/30)예측에 더 긴 과거 맥락. 단 샘플 수 감소 주의낮음

3. 추천 다음 실험

최고 baseline = world + 도로정렬(augment) 위에서:
  • [A] 인코더 대결 (아이디어 1): GRU vs LSTM vs Mamba vs Transformer — 같은 ego 도로정렬 입력. 가장 값싸고, 원래 Mamba 관심을 정당한 무대에서 결판.
  • [B] 다중 스케일 + 어텐션 (2·3): 인코더 승자 위에 얹어 추가 향상 + 해석 그림.
  • [C] 고차 미분 피처 (4): 도로정렬 채널에 ΔΔρ·jerk·곡률 추가 — 즉시 가능.
주의: 이 방향(ego 시계열 강화)은 관계(edge) novelty가 아니다. 논문 정체성이 "temporal representation for behavior prediction"으로 이동한다. edge는 정직한 음성 ablation으로 포함.
